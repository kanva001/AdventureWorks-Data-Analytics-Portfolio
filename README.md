# AdventureWorks Inventory Accuracy & ICQA Analytics Program

## Business Context
This program simulates an enterprise inventory analytics initiative focused on accuracy,
data quality, and operational visibility—aligned with ICQA-style analysis supporting
warehouse and fulfillment operations.

## Problem Statement
Operational teams require trusted inventory metrics to:
- Detect quantity mismatches and high-risk items
- Identify variance patterns by product and location
- Prevent downstream planning and fulfillment defects

Inconsistent data quality and limited visibility increase operational risk.

## Solution Overview
An end-to-end analytics solution is delivered incrementally using Agile sprints:
- Excel-based KPI delivery for rapid operational visibility
- SQL-based analytics for scalable, repeatable metrics
- Data quality validation to protect metric integrity
- Power BI dashboards for stakeholder-ready reporting

## Technology Stack
- Microsoft SQL Server (AdventureWorks OLTP & DW)
- Excel (Reconciliation, KPI analysis)
- Power BI (Dashboards)
- GitHub (Version control, release notes, documentation)

## Delivery Model
The program follows phased delivery with sprint releases and release gates:
- Phase 0: Setup & Standards
- Phase 1: Excel ICQA Analytics
- Phase 2: SQL Analysis
- Phase 3: Data Quality & Validation
- Phase 4: Power BI Dashboards
- Phase 5: Capstone & Interview Readiness
